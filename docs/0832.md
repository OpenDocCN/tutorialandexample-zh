**答:**这种编程语言的开发方式不依赖于任何硬件，也不依赖于软件。Java 被称为平台无关语言，因为编译器编译代码。然后将编译后的代码转换成与平台无关的字节码。这段代码现在可以在多个系统上运行。这就是 Java 被称为“一次编写，随处运行”的原因。要让机器运行这个字节码，有一个条件是机器上应该安装一个运行时环境(JRE)。

## 问题:为什么 Java 不是纯面向对象的语言？

## 问题:解释 Java 中堆栈内存和堆内存的区别。

**答案:**

**堆栈内存:**顾名思义，静态内存是 Java 编程语言中用于静态内存分配的内存。此外，这个静态内存用于线程的执行。对于一个类，当一个对象被创建时，该对象的引用将包含在堆栈内存中，并且该对象将总是存储在堆内存中。对于创建的每个对象，单独的堆栈将由该堆栈内存创建。当线程被创建时，堆栈的自动分配将由我们正在使用的操作系统来完成。并不是应用程序的所有部分都使用它，这个堆栈内存只由一个执行线程使用。对于每个线程，在运行时，这个堆栈内存将被分配，这个堆栈内存是随机存取内存(RAM)中的一个物理空间。这个堆栈存储器遵循后进先出(LIFO)来访问存储器。简单地说，我们可以说内存管理遵循 LIFO(后进先出)。对于所有者线程，只有变量在堆栈内存中是可见的。这个堆栈内存主要用于存储局部变量和方法执行的顺序。这个堆栈内存的内存分配是连续分配的。当线程被创建时，这个内存将被创建。全球都可以访问。Java 虚拟机(JVM)将为每个线程创建一个单独的堆栈。一旦完成，我们就不能改变分配的内存。这种堆栈内存不像堆内存那样灵活。在这种堆栈内存中，分配和释放速度更快。这种类型的内存访问速度更快。这个内存的内存大小差不多。另一件重要的事情是我们可以通过使用 JVM 选项-Xss 来增加堆栈的大小。每当堆栈大小大于它所分配的限制时，就会自动抛出 Java.lang.StackOverFlowError。所以为了避免这个错误，我们只能使用 JVM 选项-Xss 来增加堆栈的大小。使用这种内存的优点是，与堆内存相比，成本会更低，而且是可以承受的。但是堆栈存储器的实现是困难的。使用这个堆栈内存，我们可以说它是线程安全的，因为对于创建的每个线程，每个线程都有自己的堆栈。

**堆内存:**堆内存又称动态内存；它是存储对象的内存区域。内存块分配和释放不是以固定的模式完成的。简单地说，我们可以说内存的分配是随机进行的。堆内存的实现很容易，但是与堆栈内存相比，这种内存的成本更高。该内存由所有线程共享。当 Java 虚拟机开始运行时，将创建堆内存。只要应用程序在运行，创建的堆就在对象的线程之间共享。对于本地堆栈内存，堆内存是可选的堆栈内存。在堆内存中，调用函数时会自动分配局部变量。当函数退出时，局部变量的释放是自动完成的。与堆栈内存相比，堆内存的大小更大。分配和释放速度较慢。此外，访问速度比堆栈内存慢。因为我们可以改变内存，所以堆内存是灵活的。JRE (Java 运行时环境)类和对象都存储在这个内存中。如替代方案所示，这种记忆不遵循任何顺序。内存分配是动态完成的。-Xmx 和-Xms JVM 选项用于增加或减少堆内存的大小。在堆栈内存中，我们可以增加堆栈的大小，但这里我们可以增加或减少堆内存的大小。对所有线程可见。Java.lang.OutOfMemoryError 会被 Java 虚拟机抛出，当 Java 虚拟机无法创建新的原生方法时就会出现这个异常。分配和释放由程序员在堆内存中完成。作为堆栈内存，堆内存不是线程安全的。需要正确同步的代码。

## 问:为什么 Java 不像在 C 和 C++编程语言中那样使用指针？

**回答:**作为一个初学程序员，使用指针并没有那么好，也没有那么安全。因为原因是它们非常复杂，而且它们在代码中的使用使得代码变得复杂，对程序员来说具有挑战性。Java 编程语言的主要目的是使代码简单而不复杂。在程序中使用这些指针会导致潜在的错误。如果我们在程序中使用指针，用户可以在这些指针的帮助下直接访问内存。所以，当我们在程序中使用指针时，我们不得不牺牲访问的安全性，因为指针的使用，这里出现了安全问题。为了减少内存空间的使用，Java 编程语言在其程序中不使用这种指针的概念，原因是这些指针在运行时花费更多的时间，并且在运行时也占用更多的内存空间。我们可以说，总的来说，这种 Java 编程语言没有也不支持指针的概念。在程序中使用指针使得健壮性变得更强，使得编程语言难以理解，也使得编程语言变得复杂。所以，这个 Java 在其中使用了引用的思想，而不是在语言中使用指针。所以，在 Java 编程语言中，这些指针被称为引用。这些引用的使用远不止指针的使用。指针有空值，但引用没有。

## 问题:解释 Java 中的实例变量。

**答案:实例变量:**实例变量在 Java 中可以说是非静态变量。这些实例变量定义在一个类中，但不在任何方法、块中，也不在构造函数中。

正如我们所知，每当一个类被定义时，它的实例就会被创建。该实例被称为对象。我们说，当且仅当一个类的对象(它的实例)被创建时，这个类才被实例化。此外，所有方法、块和构造函数都可以访问这些实例变量。

用最简单的方式，我们可以看到实例变量是任何方法之外的变量，但它们是一个类中必须的。

我们可以通过使用 Java 编程语言中不同类型的访问修饰符来声明实例变量，如 private、public、default 和 protected。

当一个类的对象被创建或者在类加载的时候，实例变量的初始化就完成了。

让我们用一个示例程序来理解实例变量。

```
public class AtheleteGame {
	public String athleteFullName;
	public double athleteSpeedPerHour;
	public int athleteAge;
	public int athleteIdNumber;

	public AthleteGame (name, speed, age, id) {
		this.athleteFullName = name;
		this.athleteSpeedPerHour = speed;
		this.athleteAge = age;
		this.athleteIdNumber = id;
	}
	public void athleteRuns() {
		double speed = 200;
		String name = “Usha”;
		int age = 30;
		int id = 234;
	system.out.println(“Athlete Name is” + name + “.”);
	system.out.println(“Athlete speed is”+ speed + “km/hr in the given target”);
	system.out.println(“Athlete age is” + age + “currently”);
	system.out.println(“Athlete id is” + id + “in the group);
	}
} 
```

在程序中，athleteFullName、athleteSpeedPerHour、athleteAge 和 athleteIdNumber 是实例变量。

问:解释 Java 编程语言中的浅拷贝和深拷贝。

答:浅拷贝和深拷贝是用来拷贝对象数据的几种方法中的一些。

**回答:**关于静态类重要的是在 Java 编程语言中，类不能像静态变量和静态方法一样是静态的。但是如果类是内部类，那么它可以是静态类。如果静态类是静态内部类，它的工作方式将与类的静态成员完全一样。

## 问:什么是数据封装？

**回答:**是面向对象编程概念之一。这是隐藏数据属性的概念，因为我们知道在这种 Java 面向对象编程语言中，这些数据和代码与类的对象相关联。此外，封装可以定义为将代码和数据组合成一个单元的过程。为了实现这种封装，Java 使用了类的概念。

## 问:解释 Java 编程语言中的抽象。

**回答:**它也是 Java 编程语言中面向对象的编程概念之一。这种抽象主要不是向最终用户展示实际的东西。这个概念在面向对象语言中的主要目的是隐藏内部的细节，它只向用户显示基本的功能。并且只显示用户需要知道的内容。
